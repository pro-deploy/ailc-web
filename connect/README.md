# Готовые конфигурации подключения

Здесь лежат готовые к вставке конфигурации MCP-сервера ailc под разные редакторы.
Те же конфигурации лендинг показывает в разделе «Подключение» с кнопкой копирования.

Принцип один: вы вставляете несколько строк в конфигурацию редактора, и при первом
обращении к серверу готовый бинарь скачивается сам. Собирать ничего не нужно.

## Какой файл куда

| Файл | Редактор | Куда вставлять |
|---|---|---|
| `claude-code.mcp.json` | Claude Code | `.mcp.json` в корне проекта |
| `cursor.mcp.json` | Cursor | `~/.cursor/mcp.json` |
| `vscode-copilot.json` | VS Code (агент GitHub Copilot) | палитра команд, «MCP: Open User Configuration» |
| `cline.json` | Cline (расширение VS Code) | иконка «MCP Servers», затем «Configure» |
| `docker.mcp.json` | любой редактор с ключом `mcpServers` | вариант без Node.js, через Docker |

## Два способа доставки бинаря

1. **npx** (в файлах по умолчанию): нужен Node.js 16 и выше. Обёртка `ailc-mcp` сама
   скачивает бинарь нужной платформы из релиза и запускает `ailc serve`.
2. **Docker** (`docker.mcp.json`): нужен установленный Docker. Готовый образ с вложенным
   бинарём, папка проекта монтируется в контейнер. Для Claude Code и Cursor подставьте
   абсолютный путь к проекту вместо `${workspaceFolder}`.

## Тонкость с nvm

Если Node.js установлен через nvm, редактор, запущенный из дока или меню, может не найти
`npx` в PATH. Тогда используйте вариант через Docker либо пропишите абсолютный путь к
бинарю `ailc` (его печатает установщик, по умолчанию `~/.local/bin/ailc`) с аргументом
`serve`.
